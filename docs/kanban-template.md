# Cards sugeridos para o GitHub Projects (Kanban)

> Crie um *Project (Board)* no repositório com três colunas exatas:
> **A Fazer**, **Em Progresso** e **Concluído**.
> Adicione os cards abaixo (basta colar o título e a descrição).
> O trabalho exige **no mínimo 10 cards**; a lista abaixo tem **14**.

## Coluna: A Fazer

1. **Configurar dashboard operacional**
   *Tela inicial com KPIs de entregas em tempo real. Prioridade Alta. Responsável: Equipe de UX.*

2. **Integrar API de rastreamento**
   *Consumir webhook da transportadora parceira. Prioridade Crítica. Responsável: Backend.*

3. **Documentar API pública**
   *Gerar contrato OpenAPI 3.1 para os clientes B2B. Prioridade Média. Responsável: Backend.*

4. **[Mudança de escopo] Filtros por prioridade**
   *Permitir filtrar o Kanban por prioridade da tarefa. Prioridade Alta. Responsável: Frontend.*

5. **Notificação por e-mail em tarefas atrasadas**
   *Disparar e-mail diário com cards vencidos. Prioridade Média. Responsável: Backend.*

## Coluna: Em Progresso

6. **Implementar CRUD de tarefas**
   *CRUD completo no backend com validações. Prioridade Alta. Responsável: Backend.*

7. **Criar quadro Kanban responsivo**
   *Layout em colunas com drag-and-drop futuramente. Prioridade Média. Responsável: Frontend.*

8. **Definir política de SLA**
   *Acordo de nível de serviço para entregas urgentes. Prioridade Baixa. Responsável: Produto.*

9. **[Mudança de escopo] Indicador de atraso**
   *Destacar visualmente tarefas com prazo vencido. Prioridade Crítica. Responsável: Frontend.*

## Coluna: Concluído

10. **Levantar requisitos com cliente**
    *Reunião inicial e mapa de personas. Prioridade Média. Responsável: Produto.*

11. **Configurar repositório GitHub**
    *Repositório público com Projects e proteção de branch. Prioridade Média. Responsável: DevOps.*

12. **Pipeline de CI inicial**
    *GitHub Actions executando build e testes a cada PR. Prioridade Alta. Responsável: DevOps.*

13. **Definir arquitetura inicial**
    *Stack Java + Spring Boot + JPA escolhida. Prioridade Alta. Responsável: Arquiteto.*

14. **Criar README com instruções de execução**
    *Documentação inicial do repositório. Prioridade Média. Responsável: Tech Writer.*

---

## Como criar o Project no GitHub

1. Acesse o repositório → aba **Projects** → **Link a project** → **New project** → **Board**.
2. Renomeie as colunas-padrão (To Do, In Progress, Done) para
   **A Fazer**, **Em Progresso** e **Concluído**.
3. Clique em **+ Add item** em cada coluna e cole os cards acima (um por linha).
4. Para a parte teórica/print do trabalho, use o modo *Board view* (não *Table*).
5. Após simular a "mudança de escopo" durante a apresentação, **mova um dos cards
   `[Mudança de escopo]`** entre colunas em frente do avaliador para demonstrar a adaptabilidade.
